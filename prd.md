# 제품 요구사항 정의서 (PRD) - ZenTask (마인드풀니스 / 미니멀 ToDo 앱)

본 문서는 수많은 할 일 목록에서 오는 시각적 피로감과 스트레스를 최소화하고, 사용자가 마음의 평온을 유지하며 하나씩 집중할 수 있도록 돕는 **마인드풀니스(Zen) 스타일의 ToDo 웹앱(ZenTask)** 요구사항 정의서입니다.

---

## 1. 제품 개요 (Product Overview)
*   **제품명:** ZenTask (젠태스크)
*   **목적:** 복잡하고 빽빽한 할 일 목록에서 탈피하여, 오늘 당장 집중해야 할 몇 가지 작업에만 몰입할 수 있는 차분한 환경을 제공합니다.
*   **핵심 가치:**
    *   **Less is More (단순함):** 하루 최대 3~5개의 핵심 할 일만 전면에 노출하여 뇌의 인지 과부하 방지.
    *   **Serene Aesthetics (평온한 미학):** 자연에서 영감을 얻은 부드러운 색감(샌드 베이지, 세이지 그린, 소프트 웜 화이트)과 부드러운 유기적 그라데이션 배경.
    *   **Calming Micro-interaction (치유형 인터랙션):** 완료 시 안개처럼 서서히 사라지거나 물방울처럼 녹아드는 부드러운 트랜지션.

---

## 2. 대상 사용자 (Target Audience)
*   일일 할 일 목록이 너무 길어 화면을 볼 때마다 스트레스와 피로감을 느끼는 사용자.
*   자연스러운 애니메이션, 차분한 디자인, 화이트 노이즈 등을 선호하는 힐링 스타일의 사용자.
*   이것저것 복잡한 설정 없이, 직관적이고 미니멀한 집중 도구를 원하는 사용자.

---

## 3. 핵심 기능 요구사항 (Core Functional Requirements)

### 3.1. '오늘의 집중' (Today's Focus)
*   **Top 3 핵심 집중제:**
    *   하루의 시작 시 가장 중요한 할 일 **딱 3개(최대 5개)**만 메인 화면에 등록할 수 있습니다.
    *   그 외의 자잘한 할 일들은 '보관함(Inbox)' 영역에 숨겨두어 시야에서 가립니다.
*   **단일 집중 모드 (Focus Mode):**
    *   현재 하고 있는 단 하나의 일만 전체 화면에 띄우고, 배경을 차분하게 어둡히는 모드.

### 3.2. 할 일(Task) 관리 및 비움
*   **초간편 등록:** 카테고리나 우선순위 태그를 최소화하고, 텍스트와 시간 입력만으로 빠르게 생성.
*   **부드러운 완료 (Zen Dissolve):**
    *   할 일을 체크하면 줄이 직 그어지는 대신, 항목이 파동이 일듯 번지며 안개처럼 서서히 스러지는 완료 효과 구현.

### 3.3. 휴식 및 명상 타이머 (Breath Timer)
*   집중이 끝난 후 3분 또는 5분간 호흡을 가다듬을 수 있는 심플한 서클 타이머 탑재.
*   화면의 원이 커지고 작아지는 템포에 맞춰 들숨과 날숨을 유도.

---

## 4. UI/UX 및 디자인 요구사항 (Design & Experience)

### 4.1. 비주얼 컨셉 (Visual Design)
*   **색상 조합:** 세이지 그린(bg-emerald-50/100, text-emerald-900), 샌드 베이지, 따뜻한 톤의 라이트 그레이. 텍스트는 눈이 편안한 차콜 계열.
*   **배경:** 고정된 단색이 아닌, 아주 미세하고 느리게 움직이는 그라데이션 배경(Slow-flowing Fluid Background)을 적용하여 심리적 안정감을 제공.
*   **타이포그래피:** 가독성이 높고 단정한 산세리프 폰트 (예: Inter 또는 Outfit) 적용.

### 4.2. 마이크로 인터랙션
*   마우스 커서의 움직임에 따라 배경의 빛이나 물결이 은은하게 반응하는 효과.
*   모든 트랜지션은 급격하게 변하지 않고 최소 0.5초 이상의 부드러운 이지인/아웃(Ease-in/out) 적용.

---

## 5. 데이터 아키텍처 및 저장 방식 (Data Architecture)
추후 클라우드/상용 DB(예: PostgreSQL, MySQL 등) 연결 및 동기화를 원활하게 수행할 수 있도록 설계합니다.

*   **클라우드 DB 적용 (Neon DB):**
    *   안정적인 데이터 저장 및 확장을 위해 서버리스 PostgreSQL 서비스인 **Neon DB**를 데이터베이스로 활용합니다.
*   **ORM(Object-Relational Mapping) 활용:**
    *   SQL 코드를 직접 작성하는 대신 **Prisma** 또는 **Drizzle ORM**을 도입하여 스키마를 정의합니다.
    *   이를 통해 스키마 정의만으로 SQLite 테이블이 자동 생성되며, 추후 환경 변수 설정값 변경만으로 PostgreSQL 등의 상용 DB로 쉽게 스위칭할 수 있도록 마이그레이션 호환성을 확보합니다.
*   **Next.js API Routes / Server Actions:**
    *   클라이언트(프론트엔드)에서 DB에 직접 접근하지 않고, Next.js의 백엔드 레이어(API Routes 또는 Server Actions)를 거쳐 DB 안전하게 데이터를 CRUD 처리합니다.

---

## 6. 개발 기술 스택 (Tech Stack)
*   **Framework:** Next.js (App Router 기반)
*   **Language:** TypeScript
*   **Database:** Neon DB (Serverless PostgreSQL)
*   **ORM:** Prisma 또는 Drizzle ORM (DB 이식성 확보 목적)
*   **Styling:** **Tailwind CSS v3** (유틸리티 퍼스트 클래스를 활용한 반응형 UI 및 모던 디자인 시스템 구축)
*   **State Management:** React Context API (타이머 및 전역 상태 관리)
